docCommand 及相关组件 样式优化方案
目标：统一使用 settingsMixin（适配 my.vue 设置）、深色模式、主题色响应、
      生产行业风格、更简约流畅

═══════════════════════════════════════════════════════════════════════════════
一、pages/wms/docOpt/docCommand.vue —— 核心改动
═══════════════════════════════════════════════════════════════════════════════

【1】模板根元素：增加 darkClass / themeClass
--------------------------------------------------------
<view class="page" :class="[sizeClass]">
改为：
<view class="page" :class="[sizeClass, darkClass, themeClass]">

同样 popup 根元素也要加上：
<view class="sn-select-popup" :class="[sizeClass]">
改为：
<view class="sn-select-popup" :class="[sizeClass, darkClass]">

<view class="unit-select-popup" :class="[sizeClass]">
改为：
<view class="unit-select-popup" :class="[sizeClass, darkClass]">


【2】script：替换 mixin 和 import
--------------------------------------------------------
删除：
import styleManager from '@/common/styleManager.js';

新增：
import settingsMixin from '@/common/settingsMixin.js';

mixins: [styleManager.styleMixin],
改为：
mixins: [settingsMixin],

删除 data 中的 iconSize / starSize / clearIconSize（这些由 settingsMixin 的 themePrimary 等 computed 替代）

删除 methods 中的 updateIconSizes() 和 onStyleSizeChanged()（settingsMixin 自动处理）

删除 watch 中的 currentSize（不再需要）


【3】模板中的硬编码颜色 → 动态主题色
--------------------------------------------------------
所有 color="#1677ff" 改为 :color="themePrimary"
所有 color="#ffa940" 保留（固定色用于星标）
所有 color="#9c9c9c" 保留（灰色图标）
所有 background: #1677ff 改为 :style="{ background: themePrimary }"
所有 border-color / box-shadow 中用到 #1677ff 的改为动态绑定

示例：
.scan-btn 的 background: rgba(22,119,255,0.1) → :style="{ background: themeAccent }"
.submit-btn background-color: #1677ff → :style="{ background: themePrimary }"
.cmd-button-item background: #1677ff → :style="{ background: themePrimary }"

三、components/SearchSelector.vue —— 搜索选择器组件
═══════════════════════════════════════════════════════════════════════════════

这个组件没有使用 mixin，需要增加 props 接收主题信息或使用 CSS 变量。
推荐方案：通过 props 接收 theme-primary 颜色，保持组件独立性。

【1】新增 props
--------------------------------------------------------
themeColor: { type: String, default: '#1677ff' },
themeBg:    { type: String, default: 'rgba(22,119,255,0.05)' },
darkMode:   { type: Boolean, default: false }

【2】模板绑定动态色
--------------------------------------------------------
所有 color="#1677ff" → :color="themeColor"
选中背景 rgba(22,119,255,0.02) → :style="isSelected(item) ? { background: themeBg } : {}"

【3】根元素加 class
--------------------------------------------------------
<view class="search-selector" :class="{ 'theme-dark': darkMode }">

【4】SCSS 增加深色
--------------------------------------------------------
.theme-dark {
  .selector-trigger {
    background: #1e1e36;
    border-color: #2a2a45;
    &.disabled { background: #252540; border-color: #2a2a45; }
  }
  .trigger-text { color: #e0e0e0; &.placeholder { color: #555; } }

  .popup-container { background: #1a1a2e; }
  .popup-header {
    border-bottom-color: #2a2a45;
    .popup-title { color: #e0e0e0; }
    .search-box { background: #1e1e36; }
    .search-input { color: #e0e0e0; }
  }

  .option-item {
    border-bottom-color: #2a2a2a;
    &:active:not(.disabled) { background: #1e1e36; }
    &.disabled { background: #1a1a2e; }
  }
  .option-name { color: #e0e0e0; }
  .option-desc { color: #666; }
}

【5】在 docCommand.vue 中使用时传参
--------------------------------------------------------
<search-selector
  ...
  :themeColor="themePrimary"
  :themeBg="themeAccent"
  :darkMode="appSettings.darkMode === 'dark'"
/>
