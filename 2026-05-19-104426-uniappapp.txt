  2. 样式与逻辑分离

  三层分离架构：

  ┌─ 页面层 (pages/)      ← 只写业务逻辑，组合组件
  │   混入 styleMixin      ← 自动获得 currentSize / sizeClass
  │
  ├─ 组件层 (components/)  ← 纯 UI，引用 CSS 变量
  │   引用 var(--color-*)  ← 从不硬编码颜色/字号
  │   混入 useSelector     ← 逻辑由 mixin 提供
  │
  └─ 基础设施层 (common/)
     ├─ theme.scss         ← CSS 变量 + SCSS Mixin
     ├─ styleManager.js    ← 运行时尺寸切换
     ├─ selectorConfig.js  ← 选择器类型定义
     └─ useSelector.js     ← 通用逻辑（数据/缓存/搜索/分页）

  关键变化：
  - 组件样式不再写 color: #333，而是 color: var(--color-text)
  - 不再写 font-size: 26rpx，而是 font-size: var(--font-md)
  - 切换字号大小（小/中/大）时，body.size-small / body.size-large 驱动所有 CSS
  变量自动更新，不需要 $forceUpdate
  - UnifiedSelector 只有 ~220 行（vs CommonSelector 的 898 行）

  4. 性能优化建议

  A. 列表渲染优化（最关键）

  当前问题：realTimeInventory.vue 的表格数据大量使用 v-for 且无 :key 优化

  建议：

  <!-- ❌ 当前 -->
  <view v-for="(item, index) in summaryList" :key="index" class="table-row">
    <!-- index 作为 key 导致数据变化时全部重渲染 -->

  <!-- ✅ 优化后 -->
  <view v-for="item in summaryList" :key="item.id || item.uniqueKey" 
  class="table-row">

  大列表虚拟滚动：当选项超过 200 条时，在 UnifiedSelector 的 scroll-view 中使用
  v-if 分段渲染，只渲染可视区域 ± 缓冲区的 20 条：

  // 在 useSelector mixin 中添加
  computed: {
    visibleSlice() {
      // 配合 scroll-view 的 scrollTop 计算
      const start = Math.max(0, this._scrollIndex - 10)
      const end = Math.min(this.filteredList.length, this._scrollIndex + 30)
      return this.filteredList.slice(start, end)
    }
  }

  B. 弹窗动画优化

  // 在 UnifiedSelector 弹窗上加硬件加速
  .us-popup {

  ---
  数据流架构

  用户操作 my.vue
      │
      ▼
  appSettings.set('themeColor', 'green')
      │
      ├─► uni.setStorageSync('app_settings', ...)   ← 持久化
      ├─► document.body.classList.add('theme-green') ← H5 端 CSS 变量即时切换
      └─► uni.$emit('appSettingsChanged', { key, value })
              │
              ▼
      settingsMixin._onSettingsChanged()   ← 所有未卸载页面自动收到
              │
              ├─► this.appSettings = { ...新的值 }
              ├─► this.themeClass = 'theme-green'
              └─► this.$forceUpdate()      ← 页面即时刷新

  关键点：页面通过 CSS 变量引用颜色/字号（如 color: var(--color-primary)），当
  body.theme-green 被切换时，所有页面的主色立刻变为绿色，不需要逐个页面刷新。

  ---
  其他页面如何响应配置

  在任意页面的 script 中加入一行：

  import settingsMixin from '@/common/settingsMixin.js';

  export default {
    mixins: [settingsMixin],  // ← 加这一行即可
    // ...
  }

  页面自动获得：
  - this.appSettings —— 当前全部设置（可直接绑定到模板）
  - 任何页修改设置 → 本页面自动刷新
  - this.sizeClass / this.themeClass / this.darkClass —— 用于根元素 class

  模板中根元素只需：

  <view class="page" :class="[sizeClass, themeClass, darkClass]">

  页面就能跟随所有外观设置实时变化。样式使用 CSS 变量书写即可（如
  var(--color-primary), var(--color-bg-page)）。
